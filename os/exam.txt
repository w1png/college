1.	Дать определение Операционная система, перечислить виды и отличия ОС.
Операционная система - программное обеспечение, обеспечивающее управление ресурсами компьютера, а также обеспечивающее взаимодействие пользователей с компьютером.

2.	Перечислить основные задачи операционной системы.
Основные задачи операционной системы:
-	отслеживание и управление ресурсами компьютера;
-	предоставление пользователю интерфейса для взаимодействия с компьютером;
-	предоставление программам интерфейса для взаимодействия с компьютером;
-	предоставление программам интерфейса для взаимодействия с внешними устройствами;
-	предоставление программам интерфейса для взаимодействия с другими программами;
-	предоставление программам интерфейса для взаимодействия с сетью;

3.	Понятие Ядро, перечислить основные задачи.
Ядро - часть операционной системы, которая отвечает за управление ресурсами компьютера. 
Основные задачи ядра:
- Обработка прерываний;
- Управление процессами;
- Ввод-вывод данных;
- Поддержка файловых систем;
- Управление памятью;

8.	Дать определение Управляющая программа.
Управляющая программа - программа, которая управляет работой других программ.
Это может быть операционная система, которая управляет работой других программ, или программа, которая управляет работой других программ.

9.	Дать определение Пользовательский интерфейс и Программный интерфейс, назвать отличия.
Пользовательский интерфейс - интерфейс, который предоставляется пользователю для взаимодействия с компьютером. 
Программный интерфейс - интерфейс, который предоставляется программе для взаимодействия с компьютером.

13.	Дать определение Файловая система.
Файловая система - программное обеспечение, которое управляет файлами и
дисками. Это позволяет пользователям создавать, удалять, перемещать, копировать и переименовывать файлы и директории.

15.	Дать определение Файл.
Файл - это набор данных, который хранится на диске. Файлы могут быть текстовыми, бинарными или другими типами.


20.	Перечислить и описать структуры каталогов.
Структура каталогов - это иерархическая структура, которая используется для организации файлов и директорий.
Структура каталогов:
-	Корневой каталог - это каталог, который содержит все остальные каталоги и файлы.
-	Домашний каталог - это каталог, который содержит файлы и директории, которые принадлежат пользователю.
-	Временный каталог - это каталог, который содержит файлы, которые создаются программой, но не сохраняются после завершения работы программы.
-	Каталог настроек - это каталог, который содержит файлы настроек программ

21.	Описать структуру папок в Windows.
Структура папок в Windows:
-	Windows - это папка, которая содержит файлы, необходимые для запуска Windows.
-	Program Files - это папка, которая содержит файлы программ.
-	Program Files (x86) - это папка, которая содержит файлы программ для 32-разрядных программ.
-	ProgramData - это папка, которая содержит файлы программ.
-	Users - это папка, которая содержит файлы пользователей.
- temp - это папка, которая содержит временные файлы.
- System Volume Information - это папка, которая содержит файлы, которые используются для восстановления системы.

22.	Дать определение Кластеры.
Кластер - это набор секторов, которые хранятся на диске. Кластеры используются для упрощения процесса чтения и записи данных на диск.

23.	Дать определение файловая система FAT, перечислить преимущества и недостатки.
FAT - это файловая система, которая используется в Windows 95, Windows 98, Windows ME, Windows NT, Windows 2000, Windows XP, Windows Vista, Windows 7, Windows 8, Windows 10.
Преимущества:
-	Поддерживается множеством операционных систем и устройств.
Недостатки:
-	Не поддерживает файлы больше 4 Гб.

24.	Дать определение файловая система NTFS, перечислить преимущества и недостатки.
NTFS - это файловая система, которая используется в Windows NT, Windows 2000, Windows XP, Windows Vista, Windows 7, Windows 8, Windows 10.
Преимущества:
-	Поддерживает файлы больше 16 эксабайт.
- Поддержка разрешения на файлы и папки.
- Поддержка зашифрованных файлов и папок.
Недостатки:
-	Не поддерживается многими операционными системами и устройствами. Linux и Mac OS X не поддерживают NTFS.
- Ограничения в встроенном сжатии файлов.

30.	Описать принцип работы стратегии доступа к памяти «Наиболее подходящий участок».
Наиболее подходящий участок - это стратегия доступа к памяти, которая использует алгоритм поиска наиболее подходящего участка памяти для размещения нового процесса. Алгоритм поиска наиболее подходящего участка памяти для размещения нового процесса основан на поиске наименьшего свободного участка памяти, который может разместить новый процесс.

31.	Описать принцип работы стратегии доступа к памяти «Наименее подходящий участок».
Наименее подходящий участок - это стратегия доступа к памяти, которая использует алгоритм поиска наименее подходящего участка памяти для размещения нового процесса. Алгоритм поиска наименее подходящего участка памяти для размещения нового процесса основан на поиске наибольшего свободного участка памяти, который может разместить новый процес.

32.	Описать принцип работы стратегии доступа к памяти «Первый подходящий участок».
Первый подходящий участок - это стратегия доступа к памяти, которая использует алгоритм поиска первого подходящего участка памяти для размещения нового процесса. Алгоритм поиска первого подходящего участка памяти для размещения нового процесса основан на поиске первого свободного участка памяти, который может разместить новый процесс.

34.	Дать определение дефрагментация.
Дефрагментация - это процесс сборки свободных участков памяти в один большой свободный участок памяти.

35.	Дать определение фрагментация.
Фрагментация - это процесс разбиения свободного участка памяти на несколько маленьких свободных участков памяти.
